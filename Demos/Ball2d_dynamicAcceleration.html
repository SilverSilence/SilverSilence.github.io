Unlike 'static acceleration' the dynamic type allows to specify a desired acceleration which will be continuosly be added to the speed for as long as you're outside the 'zone'. The ball gets very fast very quickly. It is that not very useful for general purposes, especially when it comes to tasks where short and precise movements are required.

It would probably a useful setting to either limit the speed or to apply a 'friction' coefficient to decrease the effect of the acceleration into something more userfriendly. But at the moment, both of these options aren't implemented. If they were, it is yet to be considered, if that wouldn't allow for a completely new movement type instead?
